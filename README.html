Corporate AI Agent System

A multi-agent AI system for Brooker Group committee operations. Phase 1 delivers the Capital Allocation & ALCO Committee (CAC) end-to-end loop and HR. Phase 2 rolls out 9 more departments (Finance, Risk, Legal, IT, Comms, IC, CIO, VCC, IB) on a shared onboarding framework.

In plain terms: each department gets an AI assistant that reads its Slack channels and documents, answers questions with citations, and can propose spreadsheet edits — but a human always approves before anything real changes.

Status (2026-05-13): Phase 1 Stages 1–9 complete · Phase 1 closeout (UAT + go-live) pending · Phase 2 framework (Stage 10) live · Stages 11–19 scaffolded.

How it works

The core idea: agents work on a read-only copy of corporate data and can never change the originals. Anything they want to change is parked as a proposal until a department head clicks "approve".

Tech stack — plain English

The technologies below are the building blocks of the system. Each card explains, in everyday language, what the technology does and why we use it here.

AI & reasoning Data & memory Infrastructure Communication

Qwen3.5 (the LLM)

AI & reasoning

The large language model — the "brain" that reads questions and writes answers. We run a big 122-billion-parameter version for thinking and a smaller 9B version for search.

Like the expert employee who actually reads the documents and drafts the reply.

vLLM

AI & reasoning

The engine that runs the Qwen model fast and efficiently on our own hardware, so no data ever leaves the building.

Like the office the expert works in — purpose-built so they can answer many people quickly.

DGX Spark

Infrastructure

The two physical AI computers (Spark A and B) on-site that run the models. Everything stays local — nothing is sent to an outside cloud.

Like the company car park — owned by us, parked on our property.

LangGraph

AI & reasoning

Organises each agent's thinking into clear, repeatable steps — "look it up, check confidence, draft a proposal, hand off" — instead of one messy guess.

Like a checklist or flowchart the expert follows for every task.

RAG (Retrieval-Augmented Generation)

AI & reasoning

The method of finding the right documents first, then answering from them. This is why answers come with citations and don't get made up.

Like a researcher who pulls the exact files before writing the memo.

LlamaIndex

Data & memory

Prepares documents for search — slices long files into bite-sized pieces and watches folders for new material to add automatically.

Like the assistant who indexes and files every new document as it arrives.

Qdrant (vector store)

Data & memory

A special search database that finds information by meaning, not just exact keywords — so "cash buffer" can match "liquidity reserve".

Like a librarian who understands what you mean, not just what you typed.

Embeddings

Data & memory

The behind-the-scenes step that turns text into numbers representing its meaning, so the system can compare ideas mathematically.

Like giving every paragraph a "meaning fingerprint" for fast matching.

Obsidian vault

Data & memory

Each department's "second brain" — a folder of human-readable notes, meeting minutes and decisions that the agent keeps watching and learning from.

Like the team's shared notebook of how things are done here.

PostgreSQL

Data & memory

The main records database — it stores conversation history, proposals, approvals and audit logs in a structured, reliable way.

Like the official filing cabinet that never loses a record.

MinIO

Data & memory

The store for whole files — the uploaded PDFs, spreadsheets and documents themselves live here.

Like the company document warehouse.

Slack Bolt

Communication

Connects the agents to Slack so staff can simply ask questions in their normal department channels and get answers there.

Like the agent having its own Slack account on the team.

Email notifier

Communication

Sends formal emails to department heads when a proposal needs their approval — with a "Review Now" button that opens the approval screen.

Like the assistant who emails "please sign here" with the file attached.

Approval UI

Communication

A web page where a department head sees exactly what would change (old value vs. new value) plus the evidence, then approves, edits or rejects.

Like a contract review screen — see the redline, then sign or send back.

FastAPI + Uvicorn

Infrastructure

The toolkit each service uses to receive and answer requests over the network quickly and reliably.

Like the reception desk and phone line for each department.

Docker Compose

Infrastructure

Packages every service into a sealed "container" and starts them all together. It also enforces the read-only rule on corporate data.

Like shipping containers — each service self-contained, all loaded with one command.

nginx

Infrastructure

A traffic director that spreads AI requests evenly across the two Spark computers so neither gets overloaded.

Like a host seating guests across two dining rooms to keep waits short.

Pydantic

Infrastructure

Checks that data has the right shape and values before it is used — e.g. a proposal must have a cell, an old value and a new value.

Like a form that refuses to submit until every required box is filled correctly.

Paperclip

Infrastructure

The shared "worker shell" that runs and coordinates department agents, used as the common framework for the Phase 2 rollout.

Like the standard onboarding kit every new department agent plugs into.

Prometheus + Grafana

Infrastructure

Prometheus collects health and usage numbers; Grafana turns them into live dashboards so the team can see the system is running well.

Like the dashboard gauges and warning lights in a car.

reflection-engine

AI & reasoning

Runs each night to review the day's activity, promote useful lessons into long-term memory, and suggest improvements to the agents' skills.

Like an employee reviewing their day and updating their personal playbook.

wiki-compiler

Data & memory

Turns scattered events and messages into tidy, structured wiki articles in the Obsidian vault, so knowledge stays organised.

Like a journalist writing up loose notes into a clean article.

eval-framework

AI & reasoning

A built-in test suite that re-checks known questions to make sure agent quality doesn't slip after updates.

Like a recurring exam that confirms the agent still knows its job.

Tech stack — technical

  • LLM: Qwen3.5 122B Q8 via vLLM on dual DGX Spark (nginx least-connections LB)
  • Embeddings: Qwen3.5 9B via vLLM (Spark A only)
  • Agents: LangGraph 0.2+ with PostgresSaver checkpointer
  • RAG: LlamaIndex 0.11+ chunking + Qdrant 1.12+ vector store
  • Per-dept second brain: Obsidian vault, watched by rag-ingestion
  • Chat: Slack Bolt (Python)
  • API services: FastAPI + Uvicorn
  • Database: PostgreSQL 16 (10 migrations)
  • Validation: Pydantic v2
  • Containers: Docker Compose
  • Worker shell: Paperclip (Node.js 20+)
